# 非功能需求文档 (NFRD)

## 1. 性能需求

### NFR-001：加载性能

| 指标 | 要求 | 测量方式 |
|-----|------|---------|
| 首屏加载时间 | < 2 秒 | 从打开页面到可交互 |
| 资源总大小 | < 2MB | 包含所有图片、音频 |
| 初始 HTML 大小 | < 50KB | 不含外部资源 |

**技术要求**：
- NFR-001-R1: 使用代码分割 (Code Splitting) 优化加载
- NFR-001-R2: 图片资源使用 WebP 格式压缩
- NFR-001-R3: 音频资源使用 MP3 格式，单文件 < 100KB
- NFR-001-R4: 使用浏览器缓存策略

---

### NFR-002：运行性能

| 指标 | 要求 | 测量方式 |
|-----|------|---------|
| 键盘响应延迟 | < 50ms | 按键到视觉反馈 |
| 动画帧率 | ≥ 60fps | 使用 Chrome DevTools |
| CPU 占用率 | < 30% | 游戏运行时 |
| 内存占用 | < 100MB | 游戏运行时 |

**技术要求**：
- NFR-002-R1: 键盘事件使用 requestAnimationFrame 处理
- NFR-002-R2: 动画使用 CSS Transform 而非 top/left
- NFR-002-R3: 使用 React.memo 缓存组件
- NFR-002-R4: 定时清理不再使用的游戏对象

---

### NFR-003：并发性能

| 指标 | 要求 | 测量方式 |
|-----|------|---------|
| 同时下落字母数 | 最多 10 个 | 困难难度下 |
| 轨道数量 | 5 条 | 固定 |
| 同屏元素总数 | < 50 个 | DOM 节点数 |

## 3. 可用性需求

### NFR-006：离线可用性

| 要求 | 实现方式 |
|-----|---------|
| 无网络可游戏 | 纯前端应用，无需后端 |
| 数据本地存储 | IndexedDB / localStorage |
| PWA 支持 | Service Worker 缓存 |

**技术要求**：
- NFR-006-R1: 首次加载后，后续可完全离线使用
- NFR-006-R2: 游戏数据自动保存到本地
- NFR-006-R3: 离线状态下功能完整

---

### NFR-007：数据持久化

| 数据类型 | 存储方案 | 容量限制 | 保留策略 |
|---------|---------|---------|---------|
| 游戏记录 | IndexedDB | 100 条 | 保留最近 100 条 |
| 字库数据 | IndexedDB | 50 个 | 永久保留 |
| 用户配置 | localStorage | 5MB | 永久保留 |
| 成就进度 | IndexedDB | 1 个 | 永久保留 |

**技术要求**：
- NFR-007-R1: 数据损坏时自动恢复默认值
- NFR-007-R2: 提供数据导出/导入功能
- NFR-007-R3: 存储空间不足时提示用户

---

| 要求 | 实现方式 |

**技术要求**：

### NFR-012：代码组织

| 要求 | 实现方式 |
|-----|---------|
| 模块化 | 按功能划分模块 |
| 组件化 | 单一职责原则 |
| 命名规范 | 统一命名约定 |

**文件组织规范**：
```
src/
├── components/    # UI 组件
├── engine/        # 游戏引擎
├── stores/        # 状态管理
├── utils/         # 工具函数
├── types/         # 类型定义
└── data/          # 静态数据
```

**技术要求**：
- NFR-012-R1: 单文件不超过 400 行
- NFR-012-R2: 单函数不超过 50 行
- NFR-012-R3: 嵌套深度不超过 4 层
